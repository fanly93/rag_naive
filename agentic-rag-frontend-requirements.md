# Agentic RAG 前端需求文档（完整版）

## 1. 项目概述

- 项目名称：`RAG本地知识库问答引擎`
- 项目目标：提供一个面向非技术用户的 Web 前端，实现文档上传、知识库构建、两阶段召回测试与可溯源问答体验。
- 后端能力现状：已支持文件上传、文本切分、索引构建、向量库构建。
- 系统架构定位：
  - `llama_index`：承担主要 RAG（检索与索引）能力
  - `langchain`：承担 Agent 编排，调用 `llama_index` 作为工具
  - 前端聚焦：体验、布局、状态可见性、结果可解释性

---

## 2. 页面布局与信息架构

- 顶部：仅中间标题 `RAG本地知识库问答引擎`
- 主体采用三栏布局：
  - 左栏：历史会话列表
  - 中栏：问答聊天主区
  - 右栏：知识库管理 + 召回测试
- 布局建议（桌面端）：
  - 左栏约 20%
  - 中栏约 50%
  - 右栏约 30%
- 响应式策略：
  - 宽屏保持三栏
  - 中等屏宽右栏可折叠
  - 窄屏优先显示中栏，左/右栏抽屉化

---

## 3. 顶部栏设计

- 仅显示居中标题：`RAG本地知识库问答引擎`
- 不放课程标识、按钮、额外 CTA
- 视觉上保持轻量，作为页面全局锚点

---

## 4. 左栏：历史会话模块

### 4.1 功能点

- 会话列表展示：
  - 会话标题（取首句并截断）
  - 最近更新时间
- 支持操作：
  - 新建会话
  - 切换会话
  - 删除会话（需二次确认，确认后直接删除，不可恢复）
- 支持会话搜索框（按标题关键字过滤）

### 4.2 关键约束

- 同一时刻只允许存在 `1` 个空白新会话
- 若已有空白会话，再次点击新建时提示：`请先在当前新会话中输入内容或关闭该会话`

### 4.3 状态与交互

- 当前会话高亮
- 超长标题省略号显示
- 删除确认文案：`确认删除该会话？删除后不可恢复。`

---

## 5. 中栏：聊天问答模块

### 5.1 基础能力

- 用户输入问题并发送
- 展示模型回答
- 支持加载态（消息气泡内展示思考/生成中）

### 5.2 引用可溯源展示

- 当回答基于检索结果时，在该条回答底部展示“引用片段缩略”
- 每条引用缩略展示前 `20` 字
- 点击缩略后弹出“片段详情弹窗”
- 引用不展示片段编号，不强制展示分数（分数可在右栏详情看）

---

## 6. 右栏：知识库管理与召回测试

### 6.1 空态设计

- 空态文案：`暂无知识库，请先上传文档开始构建`
- 提供主按钮：`上传文档`
- 支持格式：`.txt`、`.md`、`.pdf`
- 单次上传数量：仅 `1` 个文件

### 6.2 上传与构建向导（弹窗）

- 入口：右栏按钮触发弹窗向导
- 推荐流程：
  1. 上传文件
  2. 配置参数
  3. 提交构建

### 6.3 参数配置（用户可见）

- `知识库名称`：必填，2-50 字符
- `Chunk Size`：默认 `1024`，建议范围 `256-4096`
- `Chunk Overlap`：默认 `100`，建议范围 `0-512`，且 `< Chunk Size`
- `检索模式`（必须暴露）：
  - `Vector`
  - `Hybrid`（Vector + BM25）
- 两阶段召回参数：
  - 第一阶段 `Top-N`：默认 `20`，建议范围 `5-200`
  - 第二阶段 `Top-K`：默认 `3`，范围 `1-20`，且 `Top-K <= Top-N`
- 操作按钮：`取消` / `开始构建`

---

## 7. 状态流转与反馈机制

### 7.1 构建中

- 弹窗可最小化（非强制阻塞）
- 展示阶段进度：`上传完成 -> 切分中 -> 索引构建中 -> 向量库构建中`
- 文案示例：`正在切分并构建向量索引，请稍候...`

### 7.2 构建成功

- 提示：`知识库构建完成`
- 右栏切换到“已创建知识库”视图
- 自动激活召回测试区域
- 自动填入示例查询：`什么是 Agentic RAG 的两阶段召回？`

### 7.3 构建失败

- 弹窗文案固定：`创建向量数据库失败，请重试`
- 按钮：`确认`
- 点击确认后：右栏保持空态上传指引
- 支持保留原参数并可重试

---

## 8. 召回测试与结果展示

### 8.1 测试区

- 输入查询 + 发送按钮
- 首次未检索前，不显示结果列表

### 8.2 结果列表（默认展示）

- 默认仅展示“第二阶段重排后的最终结果”
- 每条内容包含：
  - 片段标题/文档来源
  - 相似度分数（由后端返回）
  - 正文摘要前 `100` 字 + `...`

### 8.3 初召回结果查看

- 提供下拉入口，触发弹窗显示“初召回（Top-N）结果”
- 弹窗中标识召回通道（Vector/BM25）与候选项信息

### 8.4 详情弹窗

- 点击任一结果项进入详情弹窗
- 展示：
  - 完整片段内容
  - 来源文档
  - 相似度分数
  - 命中方式（Vector/Hybrid）
- 不提供“复制片段内容”按钮（按当前需求）

---

## 9. 三栏联动规则

- 切换左栏会话时：
  - 中栏切换到该会话聊天记录
  - 右栏同步切换该会话绑定的知识库状态
- 若会话未绑定知识库：
  - 右栏显示空态上传指引
- 中栏引用片段与右栏详情弹窗样式保持一致，保证体验统一

---

## 10. 视觉规范（清新、雅致）

- 主背景：`#F7F9FC`
- 卡片/面板：`#FFFFFF`
- 主文字：`#1F2937`
- 次级文字：`#6B7280`
- 边框/分隔：`#E5E7EB`
- 强调色：
  - 信息：`#3B82F6`
  - 成功：`#10B981`
  - 提示：`#F59E0B`
- 圆角：`10-14px`
- 阴影：轻量、柔和、低对比
- 设计原则：低饱和、留白充分、分层清晰、避免厚重渐变

---

## 11. Mock 数据规范（用于联调）

- 文档标题样例：
  - `Agentic RAG 入门指南`
  - `LlamaIndex 检索实践`
  - `LangChain Agent 工具调用`
- 最终重排分数样例：`0.89 / 0.84 / 0.78`
- 文本长度策略：
  - 列表摘要：前 `100` 字 + `...`
  - 聊天引用缩略：前 `20` 字
- 初召回弹窗数据：
  - 展示 Top-N 候选
  - 标记来源通道（Vector/BM25）

---

## 12. 验收标准（MVP）

- 可完成“上传 -> 配参 -> 构建 -> 召回测试 -> 聊天引用查看”的完整闭环
- 三栏联动正确，会话切换不串数据
- 构建成功/失败状态可见且反馈一致
- 两阶段召回参数可配置且校验生效
- 新建会话“仅允许一个空白会话”约束生效
- 页面风格符合“清新、雅致”基调

---

## 13. 分阶段开发计划（Phase Roadmap）

以下 Phase 按“先骨架、再主流程、再增强体验”推进。每个阶段都可独立验收，便于迭代交付。

### Phase 0：项目初始化与设计系统底座

- 目标：搭建可持续开发的前端工程基础。
- 范围：
  - 初始化项目结构、路由、状态管理、请求层封装
  - 建立全局主题变量（颜色、圆角、阴影、字体层级）
  - 搭建三栏基础布局与顶部标题（静态占位）
- 交付物：
  - 可运行页面骨架
  - 统一 UI Token 与基础组件（按钮、输入框、弹窗、卡片）
- 验收标准：
  - 页面完成三栏占位，视觉风格符合“清新、雅致”基础方向
  - 基础组件可复用且样式一致

### Phase 1：左栏会话系统（无后端/可 Mock）

- 目标：先打通会话管理闭环，支撑后续聊天主流程。
- 范围：
  - 会话列表展示、切换、新建、删除（二次确认）
  - 会话搜索框（标题过滤）
  - “仅允许一个空白新会话”约束
- 交付物：
  - 左栏完整交互功能
  - 会话状态管理（本地或 Mock）
- 验收标准：
  - 新建/切换/删除/搜索流程无阻塞
  - 多次点击“新建”时可正确命中空白会话约束提示

### Phase 2：中栏聊天基础能力

- 目标：实现问答主路径，形成可使用的聊天体验。
- 范围：
  - 输入、发送、消息流渲染、加载态
  - 会话与消息绑定（切换会话切换聊天记录）
  - 错误提示与重试入口（基础版）
- 交付物：
  - 中栏可用聊天模块
  - 会话与消息数据模型
- 验收标准：
  - 消息交互流畅，加载态明确
  - 切换会话不串消息

### Phase 3：右栏知识库构建主流程

- 目标：完成“空态 -> 上传 -> 配参 -> 构建 -> 成功/失败”的右栏核心闭环。
- 范围：
  - 空态卡片与上传入口
  - 上传弹窗向导（单文件、格式校验：`.txt`/`.md`/`.pdf`）
  - 参数配置：知识库名称、Chunk、检索模式、Top-N、Top-K 与校验规则
  - 构建中可最小化、成功态、失败弹窗（固定文案）
- 交付物：
  - 右栏知识库构建全流程
  - 参数校验与状态机
- 验收标准：
  - 完成一次真实或 Mock 的构建闭环
  - 成功/失败分支行为与文档一致

### Phase 4：召回测试与可解释性（两阶段召回）

- 目标：实现检索结果展示与“最终结果 + 初召回”双视图能力。
- 范围：
  - 召回测试输入与发送
  - 默认展示重排后结果（100 字摘要 + 分数 + 来源）
  - 下拉入口查看初召回（Top-N）弹窗，标识 Vector/BM25
  - 片段详情弹窗（完整内容、来源、分数、命中方式）
- 交付物：
  - 右栏召回测试全交互
  - 结果与详情数据映射规范
- 验收标准：
  - 默认结果与初召回视图切换正确
  - 列表摘要长度、字段展示与需求一致

### Phase 5：三栏联动、引用链路与发布前打磨

- 目标：完成端到端体验打通，达到上线前质量要求。
- 范围：
  - 中栏回答引用缩略（前 20 字）与详情弹窗联动
  - 左中右三栏状态同步（会话切换 -> 聊天与知识库同步）
  - 成功后自动注入示例查询
  - 视觉一致性、空态文案、边界情况与异常态统一
- 交付物：
  - MVP 完整链路版本
  - 联调与回归测试清单
- 验收标准：
  - 通过本文档“12. 验收标准（MVP）”全部条目
  - 无阻塞级交互缺陷，可进入灰度或内测

### 推荐节奏（可选）

- Sprint 1：Phase 0 + Phase 1
- Sprint 2：Phase 2 + Phase 3
- Sprint 3：Phase 4 + Phase 5

如需并行开发，建议按“布局与组件组 / 聊天组 / 知识库与召回组”拆分人力，并以 Phase 3 的接口契约作为联调中线。
